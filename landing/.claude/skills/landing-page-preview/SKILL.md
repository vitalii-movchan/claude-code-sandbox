# Запуск лендинга в браузере

Только показ: запускает лендинг и снимает скриншоты. Дизайн не оценивает, код не
правит — сверку/дебаг по скриншотам делает вызывающий скилл или пользователь.

## Целевой лендинг — аргумент

Имя папки в `pages/`. **Не задан** → покажи `ls pages/` и спроси, какой именно.

## Процедура

1. **Синтаксис JS** (битый скрипт = пустая страница):
   `node --check pages/<имя>/js/main.js && node --check pages/<имя>/js/data.js`
2. **Сервер** — фоном из `pages/` (корень сервера, пути относительные):
   `cd <репо>/pages && python3 -m http.server 8000`. URL —
   `http://localhost:8000/<имя>/` (занят `8000` → другой порт).
3. **Открой** — `navigate_page` на URL (chrome-devtools MCP).
4. **Скриншоты** — `resize_page` **перед** каждым кадром (иначе ширина
   недетерминирована), затем `take_screenshot` `fullPage: true`: десктоп
   (напр. 1440×900) и мобильный (напр. 390×844 — вёрстка и бургер отличаются).
5. **Консоль** (`list_console_messages`) — ошибок быть не должно. Ожидаемо и
   безвредно: `404` на `favicon.ico` и `console.warn` от `onerror` на
   демо-картинках (`img/*.jpg`); прочие ошибки — сигнал, сообщи.
6. **Останови сервер** (`TaskStop`).
