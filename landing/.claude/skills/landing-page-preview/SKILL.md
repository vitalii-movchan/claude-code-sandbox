# Запуск лендинга в браузере

Поднимает локальный сервер, открывает лендинг в Chrome через chrome-devtools
MCP, снимает скриншоты и проверяет консоль, затем гасит сервер. Только показ —
ни дизайн не оценивает, ни код не трогает.

## Целевой лендинг — аргумент

Имя папки в `pages/` (`car_service`, `move_service`…) → работай с
`pages/<аргумент>/`. **Аргумент не задан** → покажи `ls pages/` и спроси, какой
именно (это разные изолированные лендинги, обычно имеется в виду один).

Перед запуском быстро проверь синтаксис JS — битый скрипт даст пустую страницу:

```bash
node --check pages/<имя>/js/main.js && node --check pages/<имя>/js/data.js
```

## Процедура

1. **Подними сервер** из `pages/` (это корень сервера — пути относительные,
   один сервер на все лендинги). Фоновым процессом, чтобы он не блокировал:

   ```bash
   cd "<корень-репозитория>/pages" && python3 -m http.server 8000
   ```

   URL лендинга — `http://localhost:8000/<имя>/`. Если `8000` занят — возьми
   другой порт и используй его в URL.

2. **Открой** через chrome-devtools MCP: `navigate_page` на URL лендинга.

3. **Скриншоты на двух ширинах** — задавай вьюпорт `resize_page` **перед**
   каждым кадром, не полагайся на дефолт окна (иначе десктопная ширина
   недетерминирована):
   - десктоп — `resize_page` (напр. 1440×900), затем `take_screenshot`
     `fullPage: true`;
   - мобильный — `resize_page` (напр. 390×844), затем `take_screenshot`
     `fullPage: true` ещё раз — мобильная вёрстка и меню-бургер часто
     отличаются.

4. **Консоль** (`list_console_messages`): ошибок быть не должно. Безвредны и
   ожидаемы: `404` на `favicon.ico`; `console.warn` от `onerror` на
   демо-картинках (`img/*.jpg`), пока реальные файлы не положены. Прочие ошибки
   — сигнал проблемы, сообщи о них.

5. **Останови сервер** (`TaskStop` по его task_id). Не оставляй висеть.

## Что вернуть

Скриншоты (десктоп + мобильный) и сводку консоли. Если открывал ради конкретной
проверки (сверка с референсом, дебаг секции) — её делает вызывающий скилл или
пользователь; здесь только показ. Расхождения дизайна и правки кода — вне
охвата.
