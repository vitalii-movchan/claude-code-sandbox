# Аудит слотов-изображений лендинга

Скилл отвечает на один вопрос: *какие изображения нужны этому лендингу и куда
они ложатся в `SITE_DATA`?* `<landing>` — аргумент (имя папки в `pages/`, см.
корневой `CLAUDE.md`).

Результат — таблица **`mock (slot) → ссылка для заполнения`**: слева адрес слота
в `SITE_DATA` (`reviews[0].photo`, `media.hero`, `services[2].img`…), справа
пустой плейсхолдер `← вписать` под будущую ссылку. Чем заполнять — забота
человека или соседних скиллов; этот скилл только показывает, где дырки.

## Что считается слотом-изображением

Скилл смотрит **на значение, а не на имя ключа** (ключи разнятся: `photo`,
`img`, `hero`, `avatar`…). Слот попадает в таблицу, если его строковое значение:

- начинается с `img/` (локальный путь лендинга), **или**
- имеет расширение картинки (`.jpg/.jpeg/.png/.webp/.svg/.gif/.avif`), **или**
- это `http(s)://…`-ссылка.

Обход рекурсивный, поэтому ловятся слоты на любой глубине — внутри массивов и
вложенных объектов. Адрес слота строится в JS-нотации: `a.b[0].c`.

## Граница: только диагностика

Read-only, как `landing-page-audit`: скилл **сканирует и отчитывается**, но
ничего не меняет — `data.js` не правит, картинки не достаёт и не заливает.

Перпендикулярен соседям по оси изображений:

- **получить** картинку из стока → `landing-image-source` (`search`/`fetch`);
- **залить / дать ссылку** на картинку → `landing-image-storage`
  (`upload`/`download`/`locate`);
- **показать, что и куда нужно** → этот скилл.

Типичный пайплайн: `image-audit` (что нужно) → `image-source` (взять) →
`image-storage` (залить, получить URL) → вписать URL в `SITE_DATA` вручную.
Привязку к слотам сам скилл **не делает** — он лишь печатает их адреса.

## Процедура

1. **Определи `<landing>`** из запроса. Не задан → спроси, какой лендинг.
2. **Запусти сканер** — `node scripts/scan.mjs <landing>`. По умолчанию он сам
   находит `pages/` (поднимается от папки скилла); переопределить —
   `--pages <dir>`.
3. **Покажи таблицу** из вывода как есть. Пустая (слотов нет, как у
   `video_alliance`) — это валидный результат, так и скажи.

## Скрипт

- `scripts/scan.mjs` — Node ≥ 18, без зависимостей. Грузит `data.js` в
  изолированном `vm`-контексте (подставляет `window`), забирает
  `window.SITE_DATA`, рекурсивно обходит и печатает markdown-таблицу. Чистое
  чтение: исходный файл не открывается на запись.
