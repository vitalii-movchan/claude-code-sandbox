# Каталог дефектов лендинга

Репертуар того, что ищет аудит, по четырём осям. На каждый дефект: **симптом**
(как выглядит), **распознавание** (grep-сигнатура или на что смотреть глазами),
**почему** (отчего это проблема). Кода-фикса здесь нет намеренно — audit
диагностирует, чинит `landing-page-refactor`; «как чинить» в отчёте даётся одной
фразой, не патчем.

Это репертуар, а не чек-бокс-лист: не каждый дефект есть в каждом лендинге, и
список не закрыт — встретив новую проблему той же природы (мёртвый код, утечка,
недоступность), фиксируй её по тому же шаблону.

## Ось 1. Дублирование внутри файла

**Почему ось важна:** повтор — это не «много кода», а несколько мест, которые
надо менять синхронно; рассинхрон порождает баги.

- **Похожие рендереры в `main.js`.**
  *Симптом:* два-три блока `.map(...)` с почти одинаковой структурой разметки,
  различие только в полях/классах.
  *Распознавание:* читай `main.js` — повторяющиеся `getElementById(...).innerHTML =
  arr.map(...)`. Сигнал: `grep -n 'innerHTML' pages/<имя>/js/main.js` — несколько
  однотипных.
  *Почему:* правка вёрстки карточки требует одинаковых изменений в N местах.

- **Селекторы с одинаковым телом в CSS.**
  *Симптом:* несколько селекторов с идентичным набором свойств.
  *Распознавание:* глазами по `styles.css` — повторяющиеся блоки свойств.
  *Почему:* дубль стилей; изменение дизайна правится в нескольких местах.

## Ось 2. CSS

**Почему ось важна:** мёртвый и противоречивый CSS — шум, который маскирует
реальные стили и растит файл.

- **Пустые селекторы.**
  *Симптом:* `.foo {}` без свойств.
  *Распознавание:* `grep -rn '{[[:space:]]*}' pages/<имя>/css/*.css`
  *Почему:* мёртвый код, ничего не делает, мешает чтению.

- **Мёртвые правила.**
  *Симптом:* селектор есть в CSS, но класса/элемента нет в HTML и JS не добавляет.
  *Распознавание:* для подозрительного класса — `grep -rn 'класс' pages/<имя>/index.html pages/<имя>/js/`.
  Пусто → правило мёртвое.
  *Почему:* стиль, который никогда не применяется.

- **Конфликты свойств.**
  *Симптом:* одно свойство переопределяется в том же селекторе (`font-style:
  normal` затем `italic`).
  *Распознавание:* глазами — дублирующиеся свойства в одном блоке.
  *Почему:* одно из значений мёртвое; неясно, что задумано.

- **No-op свойства.**
  *Симптом:* свойство без эффекта в контексте (`gap: 0` на не-flex/grid;
  `z-index` без `position`).
  *Распознавание:* глазами — свойство, противоречащее раскладке элемента.
  *Почему:* вводит в заблуждение, выглядит значимым, но ничего не делает.

- **Неиспользуемые `:root`-токены.**
  *Симптом:* `--color-x` объявлен, но `var(--color-x)` нигде не используется.
  *Распознавание:* для токена — `grep -rn 'var(--color-x)' pages/<имя>/css/`. Пусто → мёртвый.
  *Почему:* мёртвая переменная палитры.

- **`:root` без группировки.**
  *Симптом:* токены свалены без структуры.
  *Распознавание:* глазами — длинный плоский список переменных.
  *Почему:* (косметика) тяжелее ориентироваться; группировка Colors / Typography /
  Layout / Shadows читается лучше. Низкий приоритет.

## Ось 3. JS

**Почему ось важна:** здесь живут реальные баги — то, что ломает поведение, а не
просто шумит.

- **Утечки слушателей.**
  *Симптом:* `addEventListener` навешивается повторно (в `resize`, в цикле, на
  каждый ререндер) вместо одного делегирования.
  *Распознавание:* `grep -n 'addEventListener' pages/<имя>/js/main.js` — внутри
  обработчиков/циклов.
  *Почему:* слушатели копятся, утечка памяти, дубль-срабатывания. **Реальный баг.**

- **Нет `onerror` у загрузки фонов.**
  *Симптом:* фон грузится через `new Image()` с обработкой только `onload`.
  *Распознавание:* `grep -n 'new Image\|onload\|onerror' pages/<имя>/js/main.js` —
  есть `onload`, нет парного `onerror`.
  *Почему:* при 404 картинка падает молча — ни лога, ни сигнала, отладка вслепую.
  **Реальный баг.** (Эталон с `onerror` — в `car_service`/`move_service`.)

- **Слайдер без clamp границ.**
  *Симптом:* индекс слайда меняется без ограничения диапазона.
  *Распознавание:* читай логику слайдера — `index++`/`index--` без `% length` или
  проверки границ.
  *Почему:* выход за пределы массива → пустой слайд или ошибка. **Реальный баг.**

- **Переусложнённый рендер / повторы.**
  *Симптом:* см. Ось 1 — повторяющиеся рендереры.
  *Почему:* кросс-ссылка на дубли; здесь — про сложность чтения логики.

- **`url("...")` с двойными кавычками в JS-строке.**
  *Симптом:* `'url("' + x + '")'` — двойные кавычки внутри.
  *Распознавание:* `grep -n 'url(' pages/<имя>/js/main.js`
  *Почему:* (косметика) лишние кавычки; одинарные чище. Низкий приоритет.

## Ось 4. HTML / a11y

**Почему ось важна:** доступность дешева при правке атрибутов и реально влияет на
скринридеры; семантика — на структуру.

- **Декоративная иконка без `aria-hidden`.**
  *Симптом:* эмодзи/иконка-украшение (галочка, разделитель) без `aria-hidden="true"`.
  *Распознавание:* глазами по `index.html` — эмодзи/иконки в разметке.
  *Почему:* скринридер озвучит мусор («галочка галочка галочка»).

- **Осмысленная иконка/плейсхолдер без `role`/`aria-label`.**
  *Симптом:* иконка, несущая смысл, или фото-заглушка без `role="img"` + `aria-label`.
  *Распознавание:* глазами — значимые иконки, фоновые «фото».
  *Почему:* смысл теряется для незрячих.

- **`<input>` без label.**
  *Симптом:* поле формы без связанного `<label>` (и без `aria-label`).
  *Распознавание:* `grep -n '<input' pages/<имя>/index.html` → проверь label рядом.
  *Почему:* поле неозвучено, форма недоступна.

- **Инлайн-стили.**
  *Симптом:* `style="..."` в разметке.
  *Распознавание:* `grep -rn 'style=' pages/<имя>/index.html`
  *Почему:* стиль вне CSS, дубль, тяжелее поддерживать.

- **Несемантичный `<div>` вместо `<ul>/<li>` и т.п.**
  *Симптом:* список однородных карточек/пунктов размечен `<div>`'ами; навигация не
  в `<nav>`; самостоятельный блок не в `<section>`.
  *Распознавание:* глазами — структура контента против тегов.
  *Почему:* теряется семантика и доступность. **Важная оговорка для отчёта:** смена
  структурного тега тянет браузерные дефолты и может задеть CSS-раскладку — поэтому
  это *улучшение со спросом*, не безусловный фикс. Помечай как «кандидат, проверить
  влияние на вёрстку», а не как однозначный баг. (У лендингов от `landing-page-builder`
  семантика уже на месте — тут обычно чисто.)

## Что НЕ дефект

Чтобы аудит не выдавал ложные срабатывания:

- **Повтор data-driven паттерна между лендингами** (`escapeHtml`, порядок скриптов,
  структура `SITE_DATA`) — норма по дизайну, лендинги изолированы. Это не
  дублирование.
- **404 на демо-картинки `img/*.jpg`** с `console.warn` от `onerror` — ожидаемо:
  реальных фото ещё нет, заглушки by design. Не баг.
- **favicon в `data:`-URI** в `<head>` — так и задумано (inline-SVG, без файла).
- **Разный дизайн, палитра, набор секций** у разных лендингов — каждый по своему
  референсу. Аудит проверяет код, не дизайн.
