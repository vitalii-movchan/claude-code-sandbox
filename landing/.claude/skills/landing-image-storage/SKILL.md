# Хранилище изображений лендинга

Хранилище решает *куда/откуда*, действие — *что*. `<landing>` — аргумент всех
действий (см. корневой `CLAUDE.md`).

## Модель: action-list × storage-list

Два списка в двух папках, перпендикулярные друг другу:

- **`actions/`** — **action-list**, общий контракт (интерфейс), по файлу на действие:
  - `upload-action` — загрузить изображение в хранилище;
  - `download-action` — скачать изображение из хранилища;
  - `locate-action` — ссылка на изображение / список изображений.
- **`storages/`** — **storage-list**, реализации контракта, по файлу на хранилище:
  - `vercel-storage` — Vercel Blob (REST API);
  - `local-storage` — файловая система.

Каждое хранилище **наследует** интерфейс, но **реализует** каждое действие по-своему =
набор секций, по секции на действие. Контракт читай первым (`actions/`), чтобы
знать, чего ждать от любой реализации; обзор реализаций — `storages/storage-list.md`.

## Диспетчеризация: (action, storage) → секция

Запрос задаёт **пару** `(action, storage)`. Скилл открывает файл хранилища и
читает одноимённую секцию:

```
storages/vercel-storage.md   →  ## upload-action / ## download-action / ## locate-action
storages/local-storage.md    →  ## upload-action / ## download-action / ## locate-action
```

Например `(upload, vercel)` → `storages/vercel-storage.md` § `upload-action`.
Никакого кода-роутера: пара имён прямо адресует секцию в файле хранилища.

## Хранилище по умолчанию

Хранилище **не задано** → не угадывай молча: спроси, какое (vercel или local).
Если по контексту очевидно (упомянут Vercel/Blob/URL → `vercel`; путь в `pages/`
или «локально» → `local`) — выбери и скажи, какое взял.

## Процедура

1. **Определи `(action, storage)`** из запроса. Хранилище неясно → § «Хранилище по
   умолчанию».
2. **Прочитай интерфейс** — `actions/<action>-action.md` (контракт действия;
   общие правила — `actions/action-list.md`).
3. **Прочитай реализацию** — секцию `## <action>` в `storages/<storage>.md`.
   Там команда запуска нужного скрипта, аргументы и что вернётся.
4. **Выполни** — запусти скрипт из `scripts/<storage>/`. Скрипты исполняемые,
   запускаются по месту; SKILL.md и хранилище говорят когда какое и с чем.
5. **Отчитайся** результатом по контракту (URL после upload, путь после
   download, список при locate).

## Скрипты

- `scripts/vercel/{upload,download,locate}.mjs` — Node, REST API Vercel Blob,
  токен через `BLOB_READ_WRITE_TOKEN` в env (зависимостей-пакетов не требует).
- `scripts/local/{upload,download,locate}.sh` — bash поверх файловой системы.

Добавить новое хранилище = новый `storages/<name>-storage.md`, реализующий те же
три секции, плюс его скрипты в `scripts/<name>/`. Контракт в `actions/` при
этом не меняется — это и есть смысл общего интерфейса.

## Границы

Только изображения и хранилища: привязку к слотам (`SITE_DATA`) скилл **не делает**.
Операции адресуют один `<landing>`; соседние не трогаем.
