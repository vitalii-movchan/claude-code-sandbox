# download-action — контракт

Скачать изображение из **твоего хранилища** (Vercel Blob, локальная ФС — по
выбранному хранилищу) на диск. Это возврат **уже залитой тобой** картинки по её локатору — не
путать с `fetch` у `landing-image-source` (тот тащит **новую** картинку из
**внешнего стока**). Общие правила интерфейса — в [`action-list.md`](action-list.md).
Реализации — секция `## download-action` в каждом файле `storages/<storage>.md`.

- **Вход:** `landing`, `locator` (локатор из `upload`/`locate` — URL или путь/имя),
  опц. `dest` (куда сохранить; по умолчанию — текущая папка / `pages/<landing>/img/`).
- **Выход (stdout):** путь к сохранённому локальному файлу.
- **Инвариант:** результат — валидный файл изображения, байт-в-байт исходный.
