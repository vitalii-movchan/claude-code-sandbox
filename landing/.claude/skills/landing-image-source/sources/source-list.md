# source-list — реализации (обзор)

Источник — реализация интерфейса [`actions/`](../actions/action-list.md) поверх
конкретного стока. Каждый источник наследует action-list как контракт, но
реализует каждое действие по-своему. Источник = набор секций, по секции на
действие (`## search-action` / `## fetch-action`).

Источники, по файлу на каждый:

- [`unsplash-source`](unsplash-source.md) — Unsplash через официальный API.
  Кандидат = фото с прямым URL и атрибуцией автора; ключ `UNSPLASH_ACCESS_KEY`.

## Добавить источник

Новый `<name>-source.md` здесь, реализующий **те же** секции, что перечислены
в `actions/`, плюс скрипты в `scripts/<name>/`. Контракт `actions/` при этом не
меняется — в этом и смысл общего интерфейса. Типичные кандидаты на расширение:
Pexels, Pixabay, AI-генерация — каждый встаёт отдельным файлом-источником, не
трогая остальные.
