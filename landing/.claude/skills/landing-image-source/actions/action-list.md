# action-list — интерфейс (обзор)

Общий контракт действий над получением изображений из источника. Это
**интерфейс**: каждый источник из `sources/` реализует все действия по-своему,
но соблюдает их контракты. Реализация вправе выбирать механику (какой API,
какие параметры запроса), но **не** менять контракт.

Два действия, по файлу на каждое:

- [`search-action`](search-action.md) — найти кандидатов по текстовому запросу.
- [`fetch-action`](fetch-action.md) — скачать выбранное изображение в `img/`.

## Общие правила (для всех действий)

- Первый аргумент — `<landing>` (см. корневой `CLAUDE.md`).
- Результат — изображения (`.jpg`, `.jpeg`, `.png`, `.webp`, `.avif`). Источник
  возвращает только растровые форматы, пригодные как медиа лендинга.
- Источникам нужен ключ API в окружении (какой именно — см. файл источника);
  без ключа скрипт падает с понятной ошибкой.
- При ошибке — ненулевой код возврата и сообщение в stderr; stdout зарезервирован
  под полезный результат (список кандидатов / путь к файлу), чтобы его можно
  было передать дальше (напр. результат `search` — на вход `fetch`).
- Соблюдай лицензию и атрибуцию источника — search возвращает и данные для
  кредита автору, fetch их не теряет.

Добавить действие = новый `<name>-action.md` здесь + одноимённая секция во всех
источниках. Добавить источник — см. `sources/source-list.md`.
