# search-action — контракт

Найти кандидатов-изображения в источнике по текстовому запросу, **ничего не
скачивая**. Общие правила интерфейса — в [`action-list.md`](action-list.md).
Реализации — секция `## search-action` в каждом файле `sources/<source>.md`.

- **Вход:** `landing`, `query` (текстовый запрос, напр. «coffee shop interior»),
  опц. `count` (сколько кандидатов, по умолчанию небольшое число).
- **Выход (stdout):** список кандидатов, по строке на каждый — `id`, прямой URL
  изображения (пригодный как вход `fetch`), размеры и атрибуция автора.
- **Инвариант:** ничего не пишет на диск; результат можно скормить `fetch`
  (его `id`/URL) или показать пользователю для выбора.
