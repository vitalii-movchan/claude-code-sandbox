# fetch-action — контракт

Скачать выбранное изображение из **внешнего стока** (Unsplash и т.п.) на диск, в
локальную папку лендинга. Это получение картинки **извне**, которой у тебя ещё
нет, — не путать с `download` у `landing-image-storage` (тот достаёт уже **твою**
картинку из **твоего** хранилища). Общие правила интерфейса — в
[`action-list.md`](action-list.md). Реализации — секция `## fetch-action` в каждом
файле `sources/<source>.md`.

- **Вход:** `landing`, `ref` (что скачать — `id` или прямой URL из `search`),
  опц. `dest` (куда сохранить; по умолчанию — `pages/<landing>/img/`).
- **Выход (stdout):** путь к сохранённому локальному файлу.
- **Инвариант:** результат — валидный файл изображения на диске, готовый к
  заливке в хранилище (`landing-image-storage`, `upload`) или использованию как
  медиа лендинга. Может одним шагом искать-и-качать, если `ref` — это запрос
  (см. конкретный источник), но базовый контракт — скачать по `ref` из `search`.
