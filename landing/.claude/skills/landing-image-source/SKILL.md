# Источник изображений лендинга

Источник решает *откуда брать*, действие — *что сделать* (найти / скачать).
`<landing>` — аргумент всех действий (см. корневой `CLAUDE.md`).

Перпендикулярен `landing-image-manager`: тот про **хранение** (куда положить
уже имеющуюся картинку), этот про **получение** (откуда взять новую). Граница:
`landing-image-source` кладёт файл в `pages/<landing>/img/` → дальше его можно
залить профилем `upload→vercel` из `landing-image-manager`. Целостный пайплайн:
**source → fetch → (img/) → upload**.

## Модель: action-list × source-list

Два списка в двух папках, перпендикулярные друг другу (зеркало
`landing-image-manager`):

- **`actions/`** — **action-list**, общий контракт (интерфейс), по файлу на действие:
  - `search-action` — найти кандидатов по текстовому запросу (вернуть список);
  - `fetch-action` — скачать выбранное изображение в `pages/<landing>/img/`.
- **`sources/`** — **source-list**, реализации контракта, по файлу на источник:
  - `unsplash-source` — Unsplash через официальный API.

Источник **наследует** интерфейс, но **реализует** каждое действие по-своему =
набор секций, по секции на действие. Контракт читай первым (`actions/`), чтобы
знать, чего ждать от любой реализации; обзор реализаций — `sources/source-list.md`.

## Диспетчеризация: (action, source) → секция

Запрос задаёт **пару** `(action, source)`. Скилл открывает файл источника и
читает одноимённую секцию:

```
sources/unsplash-source.md   →  ## search-action / ## fetch-action
```

Например `(fetch, unsplash)` → `sources/unsplash-source.md` § `fetch-action`.
Никакого кода-роутера: пара имён прямо адресует секцию в файле источника.

## Источник по умолчанию

Источник **не задан** → не угадывай молча: спроси, какой. Пока в source-list
один — `unsplash`; если по контексту очевидно (упомянут Unsplash/сток-фото) —
бери его и скажи, какой взял.

## Процедура

1. **Определи `(action, source)`** из запроса. Источник неясен → § «Источник
   по умолчанию».
2. **Прочитай интерфейс** — `actions/<action>-action.md` (контракт действия;
   общие правила — `actions/action-list.md`).
3. **Прочитай реализацию** — секцию `## <action>` в `sources/<source>.md`. Там
   команда запуска нужного скрипта, аргументы, нужный ключ API и что вернётся.
4. **Выполни** — запусти скрипт из `scripts/<source>/`. Скрипты исполняемые,
   запускаются по месту; SKILL.md и источник говорят когда какой и с чем.
5. **Отчитайся** результатом по контракту (список кандидатов после search,
   путь к скачанному файлу после fetch).

## Скрипты

- `scripts/unsplash/{search,fetch}.mjs` — Node ≥ 18, официальный Unsplash API,
  ключ через `UNSPLASH_ACCESS_KEY` в env (зависимостей-пакетов не требует).

Добавить новый источник = новый `sources/<name>-source.md`, реализующий те же
секции, плюс его скрипты в `scripts/<name>/`. Контракт в `actions/` при этом не
меняется — это и есть смысл общего интерфейса.

## Границы

Только получение изображений из источников: **не** хранит (это
`landing-image-manager`, профиль `upload`) и **не** привязывает к слотам
(`SITE_DATA`). Операции адресуют один `<landing>`; соседние не трогаем.
Соблюдай лицензии и атрибуцию источника (для Unsplash — кредит автору).
