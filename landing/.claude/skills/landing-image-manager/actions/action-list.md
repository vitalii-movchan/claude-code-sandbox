# action-list — интерфейс (обзор)

Общий контракт действий над изображениями. Это **интерфейс**: каждый профиль
из `profiles/` реализует все действия по-своему, но соблюдает их контракты.
Реализация вправе выбирать механику (REST, fs, что угодно), но **не** менять
контракт.

Три действия, по файлу на каждое:

- [`upload-action`](upload-action.md) — загрузить изображение в хранилище.
- [`download-action`](download-action.md) — скачать изображение из хранилища.
- [`locate-action`](locate-action.md) — ссылка на изображение / список изображений.

## Общие правила (для всех действий)

- Первый аргумент — `<landing>` (см. корневой `CLAUDE.md`).
- Работают с изображениями (`.jpg`, `.jpeg`, `.png`, `.webp`, `.svg`, `.avif`,
  `.gif`). Прочие типы — ошибка.
- Идемпотентность не гарантируется (повторный upload может перезаписать или
  создать новую версию — см. конкретный профиль).
- При ошибке — ненулевой код возврата и сообщение в stderr; stdout зарезервирован
  под полезный результат (URL / путь / список), чтобы его можно было использовать
  дальше.

Добавить действие = новый `<name>-action.md` здесь + одноимённая секция во всех
профилях. Добавить профиль — см. `profiles/profile-list.md`.
