# Превентивный справочник: дефект → как не породить

Каждый дефект, который аудит ищет постфактум, дешевле **не создавать** на этапе
генерации, чем чинить потом. При генерации семантика, `onerror`, чистый CSS —
почти бесплатны (ломать нечего); при рефакторинге смена тега уже рискует задеть
вёрстку. Этот файл — справочник вида «дефект → решение при генерации»: пробегись
по нему перед финальной сверкой, чтобы свежесобранный лендинг сразу проходил
аудит.

**Источник истины по «что это за дефект и как распознать»** — каталог скилла
`landing-page-audit` (`references/defect-catalog.md`): симптом, grep-сигнатура,
почему проблема. Здесь — только обратная сторона: **как писать сразу так, чтобы
дефекта не было**. Не дублируй сюда признаки распознавания, не превращай в
каталог.

Порядок осей — как в каталоге аудита.

## Ось 1. Дублирование внутри файла

- **Похожие рендереры в `main.js`** → если два-три списка рендерятся почти
  одинаковой разметкой, заведи **один параметризованный хелпер**, а не копипасту
  блоков. Различие выноси в аргументы (контейнер, массив, шаблон элемента):
  ```js
  function renderList(id, items, itemHtml) {
    const el = document.getElementById(id);
    if (el && items) el.innerHTML = items.map(itemHtml).join("");
  }
  renderList("services-list", data.services, (s) =>
    `<li class="service"><h3>${escapeHtml(s.title)}</h3><p>${escapeHtml(s.text)}</p></li>`);
  renderList("reviews-list", data.reviews, (r) =>
    `<li class="review"><p>${escapeHtml(r.text)}</p><p class="review-author">${escapeHtml(r.author)}</p></li>`);
  ```
  Разная разметка карточек — нормально, у каждой свой шаблон-функция; цель —
  убрать повтор обвязки (`getElementById` + `if` + `.map().join()`), не унифицировать дизайн.

- **Селекторы с одинаковым телом в CSS** → общий набор свойств — в один класс
  (или группируй селекторы через запятую), а различия добавляй отдельно. Не
  копируй один и тот же блок свойств в каждую карточку.

## Ось 2. CSS

- **Пустые селекторы** → не оставляй `.foo {}`-заготовок «на будущее». Объявляй
  селектор, когда есть что в него класть.
- **Мёртвые правила** → пиши CSS под классы, которые реально есть в `index.html`
  или которые добавляет `main.js`. Удалил блок из вёрстки — удали и его стили.
- **Конфликты свойств** → одно свойство в селекторе задаётся один раз. Не
  оставляй `font-style: normal;` выше `font-style: italic;` — последнее
  переопределение делает первое мёртвым.
- **No-op свойства** → не ставь свойство, которое не действует в этом контексте:
  `gap` — только на flex/grid-контейнере; `z-index` — только с `position`
  (отличным от `static`); `top/left` — только у позиционированного элемента.
- **Неиспользуемые `:root`-токены** → вводи переменную палитры, только когда она
  где-то через `var(...)` применяется. Не плоди `--color-x` про запас.
- **`:root` без группировки** → сразу группируй токены комментариями — Colors /
  Typography / Layout / Shadows. Это бесплатно при генерации:
  ```css
  :root {
    /* Colors */
    --color-accent: #1573e6;
    --color-text: #2b2f36;
    --color-bg: #ffffff;
    /* Typography */
    --font-base: 1rem;
  }
  ```

## Ось 3. JS

- **Утечки слушателей** → вешай обработчик **один раз**. Не навешивай
  `addEventListener` внутри `resize`, цикла или на каждый ререндер — для
  динамических элементов используй **делегирование** (один слушатель на
  контейнере, проверка `e.target` внутри), как в мобильном меню из `architecture.md`.
- **Нет `onerror` у загрузки фонов** → если грузишь фон через `new Image()`,
  обработай обе ветки: `onload` ставит фон, `onerror` логирует `console.warn` и
  (по желанию) рисует запасной фон-тон. Эталон — `car_service`/`move_service`:
  ```js
  function loadBg(el, src) {
    const img = new Image();
    img.onload = () => { el.style.backgroundImage = `url(${src})`; };
    img.onerror = () => { console.warn(`Фон не загрузился: ${src}`); };
    img.src = src;
  }
  ```
  При 404 без `onerror` картинка падает молча — отладка вслепую.
- **Слайдер без clamp границ** → индекс слайда всегда держи в диапазоне. Либо
  по кругу через модуль, либо упирай в края:
  ```js
  // по кругу:
  i = (i + 1) % slides.length;
  i = (i - 1 + slides.length) % slides.length;
  // или с упором в края:
  i = Math.min(i + 1, slides.length - 1);
  i = Math.max(i - 1, 0);
  ```
  Голый `i++`/`i--` вылетает за массив → пустой слайд или ошибка.
- **`url("...")` с двойными кавычками в JS-строке** → внутри JS-строки пиши
  `url(...)` без вложенных двойных кавычек: ``el.style.backgroundImage = `url(${src})` ``,
  а не `'url("' + src + '")'`.

## Ось 4. HTML / a11y

- **Декоративная иконка без `aria-hidden`** → эмодзи/иконке-украшению (галочка,
  разделитель, стрелка) ставь `aria-hidden="true"` прямо при вёрстке — иначе
  скринридер озвучит мусор.
- **Осмысленная иконка/плейсхолдер без `role`/`aria-label`** → иконке со смыслом
  и фото-заглушке давай `role="img"` + `aria-label="<что изображено>"`.
- **`<input>` без label** → каждое поле формы связывай с `<label>` (через
  `for`/`id`) либо давай `aria-label`. Поле без подписи форма не озвучивает.
- **Инлайн-стили** → ноль `style="..."` в разметке: всё оформление — в
  `styles.css`. Динамику (фон-картинку из `SITE_DATA`) ставь из JS, а не инлайном
  в HTML.
- **Несемантичный `<div>` вместо `<ul>/<li>` и т.п.** → подбирай тег по смыслу
  **сразу** (список → `<ul>/<li>`, навигация → `<nav>`, самостоятельный блок →
  `<section>`/`<article>`, изображение со смыслом → `<img alt>`/`<figure>`).
  **Контейнер списка тоже семантичен:** `<ul id="services-list">`, не `<div>`.
  Это та ось, где превенция важнее всего: при генерации правильный тег бесплатен,
  при рефакторинге смена тега тянет браузерные дефолты и рискует задеть раскладку
  — потому в refactor это «улучшение со спросом», а не безусловный фикс.

## Что не считать дефектом (не «чини» при генерации)

Эти вещи аудит намеренно НЕ считает дефектом — не «исправляй» их и не плоди
ложную работу:

- **Демо-заглушки фото** (`img/*.jpg` ещё нет → 404 + `console.warn` от
  `onerror`) — by design, реальные фото подкладываются позже.
- **favicon в `data:`-URI** в `<head>` — так и задумано (inline-SVG, без файла).
- **Повтор data-driven паттерна между лендингами** (`escapeHtml`, порядок
  скриптов, форма `SITE_DATA`) — норма, лендинги изолированы; это не дублирование.
