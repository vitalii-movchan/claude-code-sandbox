# CLAUDE.md

Коллекция одностраничных маркетинговых лендингов. Каждый лендинг — буквальное
воспроизведение макета-референса в виде статического сайта на чистых HTML, CSS и
vanilla JavaScript.

Сами лендинги лежат в `pages/`
Макеты-референсы — в `ui/`.

## Архитектура (одинакова у всех лендингов)

Папки лендингов самодостаточны (`index.html`, `css/`, `js/`) и строятся по единому
**data-driven** паттерну:

- `index.html` — каркас секций; контейнеры списков пустые в разметке
  (`<div id="...-list">`), наполняются в рантайме.
- `js/data.js` — источник контента: `window.SITE_DATA` с массивами под каждый
  список и путями к медиа.
- `js/main.js` — рендерит `SITE_DATA` в контейнеры + мобильное меню. Грузится
  **после** `data.js` (оба с `defer`).
- `css/styles.css` — единственная таблица стилей; палитра в `:root` (минимум
  `--color-accent`).

Дизайн (секции, классы, типографика, палитра) у каждого лендинга свой —
диктуется его референсом. Общими остаются только эти технические правила.

## Правила

- Один сервер на все лендинги.
- Полноценный origin → `iframe`-embed, `fetch`, JSON и ES-модули работают без ограничений.
- Относительные пути.

## Запуск
**Запуск через локальный HTTP-сервер**, не из файловой системы:
  `python3 -m http.server 8000` из папки `pages/` (корень сервера) → `http://localhost:8000/<папка>/`.

## Генерация лендингов

Лендинги создаёт скилл **`landing-page`** (`.claude/skills/landing-page/`) по
картинке-референсу — он источник истины по всему процессу.

Агент **`landing-page-builder`** — обёртка, которая подключает этот скилл. 

Обращайся за подробностями (референс, медиа, сверка) к скилу (`landing-page`)
Все необходимые подробности хранятся в нем.
