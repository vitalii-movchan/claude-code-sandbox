# Debate Workflow — дебаты «адвокат vs скептик»

Переиспользуемый Workflow-скрипт для оценки любого ресурса/подписки/курса/предложения.
Два агента (адвокат ЗА и скептик ПРОТИВ) ведут **прямой диалог по раундам** —
реплика каждого автоматически передаётся оппоненту, — затем нейтральный **арбитр**
сводит всё к взвешенному вердикту с условиями «стоит / не стоит».

## Как запустить

Попроси Claude: **«прогони debate-workflow по теме …»**. Claude вызовет инструмент `Workflow`
со `scriptPath` на этот файл и нужными `args`.

Минимально — достаточно строки с темой:

```
Workflow({
  scriptPath: "/home/vitaliy/www/claude-code/research/debate-workflow.js",
  args: "курс по криптовалюте в Telegram-канале @cryptoschool, https://t.me/cryptoschool"
})
```

С параметрами:

```
Workflow({
  scriptPath: "/home/vitaliy/www/claude-code/research/debate-workflow.js",
  args: {
    topic: "подписка на сигналы по криптовалюте от канала @cryptoschool",
    url: "https://t.me/cryptoschool",
    decision: "оформлять ли платную подписку на сигналы",
    rounds: 3
  }
})
```

## Параметры (`args`)

| Поле | Обяз. | По умолчанию | Описание |
|------|-------|--------------|----------|
| `topic` | да | — | Что оцениваем: название ресурса + контекст. Можно передать просто строкой вместо объекта. |
| `url` | нет | — | Ссылка на ресурс для автосбора фактов. |
| `facts` | нет | автосбор | Готовая фактура. Если задана — этап веб-поиска пропускается. |
| `rounds` | нет | `3` | Число раундов дебатов. |
| `decision` | нет | «стоит ли вступать / оформлять подписку» | Формулировка решения. |

## Как это работает (3 фазы)

1. **Сбор фактов** — агент-исследователь через веб-поиск собирает проверяемую фактуру
   (что предлагают, цена, формат, автор, отзывы, бесплатные альтернативы, маркетинговые механики).
   Пропускается, если `facts` передана вручную.
2. **Раунды дебатов** — цикл: адвокат → скептик, реплика оппонента прокидывается напрямую.
   Каждый агент возвращает `{ reply, concessions }` — видно, что именно он признал.
   В финальном раунде обе стороны формулируют взвешенную позицию.
3. **Синтез** — арбитр читает весь транскрипт и выдаёт `{ converged_on, join_if[], avoid_if[], verdict }`.

## Результат

Workflow возвращает объект:

```js
{ topic, rounds, transcript: [{round, side, reply, concessions}...], synthesis: {converged_on, join_if, avoid_if, verdict} }
```

## Замечания

- ⚠️ Это **поддержка решения**, а не финансовый/инвестиционный совет. Особенно для крипты:
  агенты работают только с найденными фактами и не проверяют легитимность/риски скама за пределами открытых данных.
- Качество вывода зависит от качества фактуры. Если веб-поиск дал мало — лучше передать `facts` вручную.
- Скрипт можно редактировать и перезапускать без пересоздания: меняешь файл → снова вызываешь `Workflow({scriptPath})`.
