# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working in this directory.

## Что это

`~/www/claude-code` — **песочница (sandbox)** для учебных и экспериментальных сlaude-проектов.
Это не самостоятельный проект, а контейнер для других проектов.

Каждая подпапка (`tic_tac_toe/`, `landing/`,`research/`, т.п.) — отдельный изолированный проект со
своим контекстом `CLAUDE.md`, **набором пакетов(скиллы, агенты, плагины и т.п.)** инструментов.

## Правила песочницы

- **Песочница (sandbox)**
    - **Песочница (sandbox)** — временное место для работы, не постоянный дом.
    - **Корень держим чистым.** Не храни здесь информацию о конкретных проектах.
    - **Единственные уместные факты на уровне корня** — этот файл `CLAUDE.md`.
- **Проектов - контекст**
    - **Контекст проекта** - живёт на уровни проекта внутри его папки и его `CLAUDE.md`.
- **Проект - изолированность**
    - **Зависимость проектов** Они не пересекаются и не делятся состоянием.
    - **Проекты изолированы** Каждый исполняется в своей изолированной среде.
- **Проект - расширения**
    - **Расширения — внутри проекта.** Каждый проект обладает своим набором скиллов, агентов, инструментов и т.п.
    - **Локальные расширения — видимость.** Локально установлены скиллы, агенты, инструменты и т.п. видимы только внутри
      конкретного проекта (контекст проекта), а не глобально на уровне пользователя (глобальный контекст).
    - **Локальные расширения — создание нового расширения.** Если проекту нужен новый скилл, агент, инструмент и т.п. —
      создавай его внутри самого проекта по умолчанию.
- **Готовый проект выносится наружу.** Завершённый проект переезжает в отдельный git-репозиторий / пакет / модуль /
  плагин / скилл-агент.

## Как работать с проектом

Открывая задачу внутри подпапки, ориентируйся на её собственный `CLAUDE.md` как на источник истины по этому проекту.

Работай внутри конкретного проекта, если задача явно про конкретный проект.
Соседние проекты не трогай, если задача явно не про них.
