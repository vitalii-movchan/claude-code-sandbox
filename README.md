# claude-code — sandbox

Песочница для учебных и экспериментальных проектов на **Claude Code**.

Это не один проект, а контейнер. Здесь я пробую идеи, отлаживаю подходы и учусь работать с инструментом. Каждая подпапка — отдельный самодостаточный проект со своим контекстом.

## Принципы

- **Каждый проект изолирован.** Свой контекст, свой `CLAUDE.md` и, при необходимости, свой набор скиллов, агентов и тулов. Проекты не пересекаются и не делятся состоянием.
- **Корень чистый.** Информация о конкретных проектах живёт внутри их папок, а не здесь. Песочница не должна превращаться в свалку.
- **Расширения — внутри проекта.** Новый скилл, агент или тул добавляется в саму папку проекта, а не в корень.
- **Готовое уезжает наружу.** Завершённый проект переносится в отдельный git-репозиторий / пакет / модуль / скилл-агент. Sandbox — временное рабочее место.

## Скиллы и агенты по проектам

Скиллы, агенты и тулы живут внутри проекта (в его `.claude/`), а не в корне песочницы — каждый проект расширяется независимо. Состав динамический, поэтому он перечислен по проектам ниже, а не сведён в таблицу.

### `landing/`

**Скиллы**

- `landing-page` — генерирует одностраничный маркетинговый лендинг по картинке-референсу: статический сайт на чистых HTML/CSS/vanilla JS без сборки и фреймворков, с data-driven контентом (`window.SITE_DATA`).

**Агенты**

- `landing-page-builder` — обёртка для создания лендинга по референсу; подключает скиллы `landing-page` и `frontend-design`. Если референс не приложен — запрашивает картинку.

### `tic_tac_toe/`

_Собственных скиллов и агентов нет — работает на встроенных возможностях Claude Code._

### `research/`

_Собственных скиллов и агентов нет — использует Workflow-скрипты (см. выше) и встроенные возможности Claude Code._
