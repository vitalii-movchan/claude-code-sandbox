# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Обзор

Репозиторий — коллекция самодостаточных одностраничных HTML-игр (крестики-нолики). Каждый файл — отдельная законченная игра: HTML, CSS и JS встроены в один `.html`, без зависимостей, сборки, пакетного менеджера и тестов. Запуск — открытием файла в браузере (двойной клик или `file://`). Сервер не требуется.

## Файлы-игры

- `heaven.html` — светлая «небесная» версия (синее небо, плывущие облака) с **намеренно слабым** ботом: вероятностные промахи (`aiMove`, ~70% берёт выигрыш/блок, иначе случайный ход).
- `dark-academia.html` — версия в эстетике dark academia (сепия, старое золото, шрифты с засечками) с **непобедимым** ботом на полном minimax.

## Общая архитектура игр

Все игры построены по одному шаблону — IIFE со `"use strict"`, человек играет `×` и ходит первым, бот — `○`. Общие строительные блоки (имена функций совпадают между файлами):

- `WIN_LINES` — 8 выигрышных линий (ряды/столбцы/диагонали) как индексы 0–8.
- `board` — массив из 9 строк (`""` / `"×"` / `"○"`); UI-ячейки генерируются скриптом в цикле и хранятся в `cells`.
- `checkWinner(b)` → `{winner, line}` или `null`; `isFull(b)`; `emptyCells(b)`.
- `place(idx, player)` ставит символ и блокирует ячейку; `onCellClick` обрабатывает ход человека, затем с задержкой `setTimeout(~420мс)` делает ход бота («размышляет…»).
- `checkEnd`, `highlight(line)`, `lockBoard`, `setStatus`, `restart` — общий поток конца партии и UI.

**Различие между играми — только в движке бота и в CSS-оформлении.** Игровая логика поля переиспользуется без изменений. При создании новой версии: копируй отлаженную логику поля, меняй движок бота (функция выбора хода, вызываемая в `onCellClick`) и палитру/разметку.

### Движки ботов

- Слабый (`heaven.html`): `aiMove` — `findWinningMove` для выигрыша/блока с вероятностью, иначе `randomEmpty`.
- Непобедимый (`dark-academia.html`): `bestMove(b)` перебирает пустые клетки, оценивая через `minimax(b, depth, isMaximizing)`. Оценка: победа ИИ `10 - depth`, поражение `depth - 10`, ничья `0` (глубина заставляет выигрывать быстрее, проигрывать позже). Поле 3×3 считается мгновенно — оптимизация не нужна.

## Соглашения

- `heaven.html` помечен `<meta name="color-scheme" content="only light">`, `dark-academia.html` — `only dark`, чтобы расширения вроде Dark Reader не инвертировали палитру. Сохраняй при правках оформления.
- Язык интерфейса и комментариев — русский.
- Палитра задаётся CSS-переменными в `:root`; меняй тему через них, а не точечными правками по файлу.

## Проверка игр

Тестов нет — проверка ручная в браузере. Для непобедимого бота: сыграть несколько партий, пытаясь выиграть (должна выходить только ничья или победа бота), проверить ловушку «угол → центр → противоположный угол», кнопку рестарта, подсветку выигрышной линии и отображение ничьей.
